# DOCX Resume Style Guide

Use this for ATS-friendly SWE resumes that should import cleanly into Google Docs.

## Typography

- Font: Arial.
- Name: 18-22 pt, bold.
- Contact: 9 pt.
- Body: 10 pt preferred; 9.5 pt acceptable for one-page pressure; avoid <9 pt.
- Section headings: 10.5-11 pt, bold, uppercase optional.
- Bullet text: 9.5-10 pt.

## Layout

- US Letter, portrait.
- Margins: 0.5-0.75 inch.
- Single column.
- Default to the user's existing resume design language when it is ATS-safe.
- No tables for layout.
- No text boxes, icons-only links, columns, photos, charts, skill bars, headers, or footers.
- Use visible URL text for LinkedIn and GitHub.
- Keep section order recruiter-friendly: Summary, Technical Skills, Experience, Open Source/Projects, Education, Certifications.
- Preserve ATS-safe separators from the source design, especially horizontal rules under the header or between major sections. Use paragraph borders instead of floating shapes.

## Spacing

- Keep line spacing near single or 1.0-1.05.
- Use tight but readable spacing after bullets.
- Match the source resume's section spacing and heading cadence before inventing a new rhythm.
- Do not accept large empty bottom margins on a one-page resume unless the content is already at the minimum truthful density.
- Prefer deleting weak bullets over compressing typography too much.
- If the page ends with large empty space, first restore missing separators, reduce excess paragraph-after spacing, and consider one additional high-value bullet before shrinking fonts.
- Keep each bullet to one core outcome: action + scope + technical method + measurable result.

## ATS Checks

- Text must be selectable and extractable.
- Reading order should match visual order.
- Important JD keywords should appear naturally in Summary, Skills, and top bullets.
- Avoid keyword stuffing and unsupported claims.
